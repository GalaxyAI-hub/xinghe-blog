# 星河博客 - 发布工作流

## 日常更新流程

### 1. 内容生成
JARVIS 根据每日热点/思考生成博客草稿：
```
posts/drafts/YYYY-MM-DD-title.md
```

### 2. 审核修改
星河审核草稿，修改确认后移至发布目录：
```
posts/YYYY-MM-DD-title.md
posts/YYYY-MM-DD-title.html
```

### 3. 更新首页
在 `index.html` 中添加新文章条目

### 4. 推送发布
```bash
cd xinghe-blog
git add .
git commit -m "新增博客：文章标题"
git push
```

## 内容规范

### 文章类型

| 类型 | 频率 | 说明 |
|------|------|------|
| 热点解读 | 1篇/天 | 基于日参提炼的深度分析 |
| 实战分享 | 2-3篇/周 | AI工具使用心得、方法论 |
| 思考感悟 | 1-2篇/周 | 超级个体成长思考 |

### 写作原则

1. **不暴露隐私**：不提及具体作息、住址、家人信息
2. **价值导向**：每篇文章要有明确的读者价值
3. **可复用**：内容可直接用于公众号/社区
4. **保持人设**：星河AI超级个体视角
